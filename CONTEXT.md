# CONTEXT — TBH_BOT (глоссарий домена)

Скрин-автопилот для Unity-оверлей-игры «TBH: Task Bar Hero». Только термины предметной
области и их точные значения. Без деталей реализации.

## Термины

- **Грейд (rank/tier)** — редкость предмета, 10 тиров: обычный < необычный < редкий <
  легендарный < бессмертный < аркана < запредельный < celestial < божественный < космический.
  Единственный надёжный источник — **слово на тултипе** (OCR). Цвет рамки НЕ источник грейда
  (путает тиры). Не прочитан/не подтверждён → **«неизв»** (никогда не выдуманный грейд).

- **Тултип** — всплывающая карточка предмета при наведении. Unity рисует её **только когда
  окно игры в фокусе (foreground)**. Без фокуса тултипа нет → грейд прочитать нельзя.
  ⇒ Перед сканом бот ОБЯЗАН вывести игру вперёд; если не вышло — скан пропускается (без
  холостого вождения мыши = «судорог»), с понятным сообщением.

- **Прескан** — стартовый осмотр: грейд-разбивка инвентаря HERO и 5 вкладок тайника.
  Информационный отчёт, не влияет на безопасность мержа.

- **Судороги** (баг-симптом) — бессмысленное дёрганье мыши по слотам, когда ховер идёт, но
  тултипы не появляются (нет фокуса) → читается «неизв». Устраняется правилом «нет фокуса —
  нет скана».

- **Деликатно / Напролом** — режимы: деликатно фармит когда юзер отошёл и уступает мыши;
  напролом фармит сразу (юзер спит / не за ПК).

- **Босс-стадия** — перед боссом мигает WARNING; высокий шанс сундука босса. Сундук надо
  поймать; если не появился за 30с после смерти босса — проверить почту.
